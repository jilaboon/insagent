InsAgent / System Architecture

שכבת האינטליגנציה של המשרד: מנתונים גולמיים, להמלצות מבוקרות, לפעולה מסודרת.

InsAgent בנויה כמערכת תפעולית מאובטחת למשרד ביטוח, שיושבת מעל קבצי המשרד, חוקי המשרד והידע המקצועי. היא לא מחליפה CRM, אלא הופכת את הנתונים לפעולה ברורה, עם בקרה אנושית, הרשאות, תיעוד ואפשרות להרחבת AI באופן מבוקר.

מסלול העבודה שהלקוח רואה

זהו הזרם העסקי הפשוט והברור שהמערכת מייצרת עבור המשרד, מהקליטה הראשונית ועד טיפול בפועל.

המערכת מסבירה בכל שלב מה קרה, למה זה חשוב, ומה הצעד הבא
1
כניסה מאובטחת
המשתמש נכנס דרך Supabase Auth. המערכת בודקת סשן תקף לפני גישה למסכים ול־API.
Login Session Protected Routes
2
קליטת נתונים
קבצים מיובאים, מזוהים, עוברים ולידציה וממופים לפרופיל לקוח אחיד עם היסטוריית יבוא.
CSV / BAFI Validation Import Jobs
3
בניית תמונת לקוח
המערכת מאחדת לקוח, פוליסות, מסלולי השקעה, דמי ניהול, מצב עדכניות ומקורות המידע.
Customer 360 Policies Freshness
4
זיהוי תובנות
חוקי משרד ותובנות מחושבות מזהים חידושים, פערי כיסוי, ריכוזים, סיכון, שירות, והזדמנויות.
Deterministic Rules Scoring
5
בדיקה ופעולה
המשרד בודק תובנות, מייצר הודעות או פעולות, מתעד החלטות, ומנהל משימות עם Audit.
Review Message Drafts Audit Trail

מפת המערכת

התרשים הבא מראה איך שכבות המערכת מחוברות: משתמשים, שכבת אבטחה, אפליקציה, מנועי החלטה, נתונים ובקרות.

שכבת גישה ואפליקציה

ממשק עברי מלא, RTL, דשבורד, לקוחות, תובנות, כללים, יבוא, משימות והודעות.

Supabase Auth למסכים ול־API
React Query לטעינה מהירה ועדכון נתונים

משתמשים פנימיים

בעלים, מנהלים, סוכנים, ותפעול. כל אחד רואה את סביבת העבודה של המשרד דרך אותה מערכת אחידה.

כניסה מאובטחת
סביבת עבודה פנימית בלבד
מסכים מותאמים לתהליך עבודה
1. Authentication & Session Guard
בדיקת משתמש מחובר לפני גישה למסכים ולקריאות API.
2. Secure API Layer
Route Handlers שמבצעים קריאה, ולידציה, Audit ובמקרים מסוימים גם Rate Limit.
3. Business Logic Layer
מנוע יבוא, מנוע התאמת כללים, מנוע תובנות, מנועי ניסוח הודעות ושירותי סניטיזציה.
4. Data & Intelligence Layer
לקוחות, פוליסות, תובנות, טיוטות הודעה, כללי משרד, Tip Library, יבוא, Audit ו־Provenance.

מנועי החלטה

שכבת אמון, בקרה ואבטחה

הלקוח צריך להבין לא רק מה המערכת עושה, אלא גם למה אפשר לסמוך עליה ואיך שומרים על שליטה.

המערכת לא “שולחת לבד”, אלא פועלת בתוך מסגרת מבוקרת
כניסה וסשן הגישה למערכת ול־API עוברת דרך Supabase Auth וסשן תקף.
Validation לפני כתיבה בקריאות רגישות מתבצעת ולידציה של payload לפני עבודה עם הנתונים.
Audit Trail פעולות קריטיות כמו יבוא ויצירת הודעות נרשמות לצורכי בקרה ותיעוד.
צמצום מידע לפני AI לפני שליחה למודלים חיצוניים המערכת מנקה מזהים אישיים ופרטים רגישים שאינם נדרשים למשימה.

גבול ברור בין “נתונים” ל“AI”

רוב המערכת מבוססת על נתוני משרד, חוקים ותהליכים. יכולות AI יושבות מעל זה רק בנקודות מוגדרות, לדוגמה ניסוח הודעה או הרחבת תובנה.

ליבת המערכת: נתונים, כללים, תיעוד, תורים, מסכים
AI: ניסוח, הצעות, הרחבה מבוקרת של Insight
Trust Layer: אימות, בקרה, Audit וצמצום מידע

מה הלקוח מקבל בפועל

לא “מערכת AI כללית”, אלא סביבת עבודה למשרד ביטוח שהופכת מידע מפוזר לתמונה אחת, עם המלצות והמשך טיפול.

למנהל המשרד

רואה איפה יש עומס, איפה יש הזדמנויות, ואילו תובנות דורשות החלטה.

לסוכן

מקבל תמונת לקוח ברורה, הסבר למה כדאי לפעול, וטיוטות עבודה מוכנות לבדיקה.

למשרד כולו

שפה אחת, תהליך אחד, ותיעוד אחד. פחות עבודה ידנית, פחות ידע “בראש”, יותר סדר, שקיפות ופעולה עקבית.

לשלב הבא

הבסיס כבר מוכן להרחבות כמו ניתוח מסמכים, AI עמוק יותר, שיפור הרשאות, ו־automation מבוקר.

עקרון בלתי מתפשר

AI תומך בהחלטה. הוא לא מחליף את שיקול הדעת של המשרד.

InsAgent היא מערכת Intelligence Layer למשרד ביטוח: מאובטחת, מוסברת, ניתנת לבקרה, ומוכנה להתרחבות.
